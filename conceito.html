Visao Geral do Produto

Dunee transforma a gestao de negocios de beleza — do caderninho ao painel inteligente.

🎯

Problema

  • Agendamentos por WhatsApp geram conflitos e no-shows
  • Controle financeiro inexistente ou em caderninho
  • Cliente nao sabe horarios disponiveis em tempo real
  • Profissional perde tempo gerenciando ao inves de atender
  • Dono do salao nao tem visao geral do negocio
  • Autonomos nao tem ferramenta profissional acessivel
💡

Solucao Dunee

  • Agendamento online 24/7 com link personalizado
  • Dashboard financeiro com receita, comissoes e despesas
  • Painel admin multi-nivel (dono, gerente, profissional)
  • Lembretes automaticos via WhatsApp
  • Pagina publica de cada estabelecimento em dunee.com.br/slug
  • Plano gratuito para autonomos, escalando para saloes

Diferenciais Competitivos

Multi-tenant Real

Cada empresa tem seus dados 100% isolados no mesmo banco via Row Level Security do PostgreSQL. Escala para milhares de tenants sem custo extra de infraestrutura.

Plano Gratis para Autonomos

Profissional autonomo usa de graca com recursos essenciais. Quando cresce e contrata equipe, faz upgrade naturalmente — funil de aquisicao organico.

Velocidade & Performance

Frontend na Vercel (CDN global, edge functions), banco Supabase com connection pooling. Pagina publica carrega em menos de 1 segundo.

WhatsApp Nativo

Integrado com WhatsApp Business API para confirmacoes, lembretes e remarketing. Reduz no-show em ate 60%.

Schema relacional com Row Level Security para isolamento multi-tenant.

Indices Criticos para Performance

-- Busca de agendamentos por profissional e data (query mais frequente) CREATE INDEX idx_appointments_tenant_prof_date ON appointments (tenant_id, professional_id, date); -- Busca de horarios disponiveis (pagina publica) CREATE INDEX idx_appointments_tenant_date_status ON appointments (tenant_id, date, status) WHERE status NOT IN ('cancelled'); -- Slug da pagina publica (unico por tenant) CREATE UNIQUE INDEX idx_tenants_slug ON tenants (slug); -- Busca de clientes por telefone (dedup) CREATE INDEX idx_clients_tenant_phone ON clients (tenant_id, phone); -- Transacoes por periodo (relatorios financeiros) CREATE INDEX idx_transactions_tenant_date ON transactions (tenant_id, date DESC); -- Bloqueios de horario ativos CREATE INDEX idx_blocked_times_tenant_prof_date ON blocked_times (tenant_id, professional_id, date);

Row Level Security (RLS)

-- Exemplo: policy para appointments ALTER TABLE appointments ENABLE ROW LEVEL SECURITY; CREATE POLICY "Tenant isolation" ON appointments USING (tenant_id = ( SELECT tenant_id FROM users WHERE id = auth.uid() )); -- Pagina publica: permitir leitura anonima filtrada CREATE POLICY "Public read by slug" ON services FOR SELECT USING ( active = true AND tenant_id IN ( SELECT id FROM tenants WHERE slug = current_setting('app.current_slug') ) );

Seguranca & Escala para Alto Trafego

Preparado para crescer de 10 para 10.000 tenants sem re-arquitetar.

Seguranca

Multi-tenant com RLS

Row Level Security no PostgreSQL garante isolamento a nivel de banco. Mesmo com bug no codigo, tenant A nao acessa dados do tenant B. Cada query filtra automaticamente por tenant_id.

Auth com JWT

Supabase Auth gera JWT com claims do tenant. Validacao no Edge Middleware do Next.js (antes de chegar no servidor). Refresh token automatico. CSRF protection built-in.

Rate Limiting

Vercel WAF para DDoS protection. Rate limit na API de agendamento (evita bot spam). Captcha no formulario publico (hCaptcha). IP blocking por abuso.

Dados Sensiveis

Env vars no Vercel (nunca no codigo). Supabase service role key so no server-side. Dados de pagamento nunca tocam nosso banco (Stripe/Pagar.me tokeniza).

Escala & Performance

CDN Global (Vercel Edge)

Paginas publicas servidas do edge mais proximo. ISR (Incremental Static Regeneration) para revalidar a cada 60s sem rebuild. Latencia <100ms no Brasil.

Connection Pooling

PgBouncer no Supabase gerencia pool de conexoes. Suporta milhares de requests simultaneos sem esgotar conexoes do PostgreSQL. Transaction mode para serverless.

Serverless Auto-scale

API Routes na Vercel escalam de 0 a milhares de instancias automaticamente. Paga por uso. Sem servidor para gerenciar. Cold start <200ms.

Cache Strategy

Pagina publica: CDN cache 60s. Dados do painel: SWR (stale-while-revalidate). Calculo de disponibilidade: cache 30s no edge. Busca de slug: cache indefinido ate mudanca.
